Backend framework

Then, around 2005, new technologies arrived to make the development process easier : Backend frameworks

Backend framework = A box located in a server in which you can find a multiple functionalities which developers can use according to their needs
Framework Language First release
Symfony PHP 2005
Laravel PHP 2011
Django Python 2005
Ruby on rails Ruby 2005
Code igniter PHP 2006
ASP.NET C# 2002
Spring Java 2004

The traditional process of a backend framework is :


  • Retrieve the data from the database
  • Process the data to extract only what we want
  • Create the HTML file based on all the data
  • Send the HTML file to the user

Benefit : You can modify everything to fit your needs

Disadvantage : But it also implies being able to program

Frontend framework

Framework Creator First release
Angular Google 2016
React Facebook 2013
Vue.js Evan You 2014

Benefit : You can quickly create a smooth application with only one language

Disadvantage : it evolves very quickly so, you need to learn new things every 2 or 3 months

Nowadays

The evolution is even faster than before. Web technologies are not only used to create website, they are used to create real applications

PWA : Progressive web app, it replicates the behavior of a native application

  • Twitter
  • Twitch
  • Youtube
  • Web version of Discord

Desktop application : It can interact with elements on the computer such as the processor, the graphic card or the storage.

You can use the framework ElectronJs to create real desktop apps.

Microsoft created Teams using this framework.

I also created a desktop app called Hashtag. You can find it here :

  • https://www.electronjs.org/apps/hashtag
  • https://github.com/MrAnyx/HashTag

Web technologies are no longer used to only create websites. They are used everywhere. The process of learning web technologies is complicated but not impossible.
